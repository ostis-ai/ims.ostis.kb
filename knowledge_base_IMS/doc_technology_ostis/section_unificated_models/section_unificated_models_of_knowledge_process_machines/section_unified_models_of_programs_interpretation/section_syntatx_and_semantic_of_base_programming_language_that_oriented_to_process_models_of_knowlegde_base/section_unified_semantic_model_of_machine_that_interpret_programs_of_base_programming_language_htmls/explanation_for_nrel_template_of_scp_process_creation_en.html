Each linking of the relation template of scp-process creation* связывает sc-node, denote an scp-program,and the set, comprising all sc-elements, for which in case of scp-process creation their unique copies will be generated, behind an exception of the sc-elements, which are caught in one of the sets,which was the second element of the appropriate linking relations variables of the scp-program*, copied constants of an scp-program*. Elements of this set are both sc-connectorsand sc-nodes. In this case it is possible situation, when sc-connector is element of this set, and incident for him sc-node isn't an element of this set. An example of this situation is inclusion in a given set sc-arc, connecting an scp-operator with operand, which is scp-constant'.

In template of scp-process creation* fall:

  1. sc-node, denoting the set of the scp-operators of the given scp-program;
  2. sc-nodes, denoting the scp-operators of the given scp-program;
  3. linking of the relation of the next operator* and more quotients;
  4. all elements of the sets that are the second and the third operands of the scp-operators of generation of sc-construction on arbitrary sample и scp-operators of search of sc-construction on arbitrary sample;
  5. all outgoing sc-arcs of accessory,which is incident listed above sc-elements, and also sc-elements, find themselves in many copied constants of an scp-program*;
  6. all entering sc-arcs of accessory,which is incident listed above to the sc-arcs of accessory;
  7. all sc-arcs of accessory, connecting sc-nodes, which denote scp-operators of the given scp-program and the main sc-nodes, which denote the class of scp-operators.
