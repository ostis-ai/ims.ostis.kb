Задачей агента поиска всех предложений конкретного разработчика является поиск всех предложений разработчикa, который является автором данных предложений. На вход данному агенту подается разработчик (экземпляр класса пользователя). Данный агент инициируется при условии появления в памяти вопросной конструкции, соответствующей действию. поиск всех предложений разработчика. Возможные результаты работы агента:

  • Если на вход агенту пришел не знак пользователя, то генерируется сообщение об ошибке (Wrong first parameter! It must be user sign(element of ui_user)).
  • Если в результате поиска предложения не были найдены, то генерируется сообщение "The search hasn't given any results".
  • Если в результате поиска предложения были найдены, то они будут выведенены. Осуществляется поиск нерассмотренных (не утвержденных и не отклоненных), утвержденных и отклоненных предложений. Пример результата работы агента представлен на рисунке. Класс действия (смотреть рисунок) должен являться ключевым sc-элементом Раздел. Предметная область действий разработчиков sc-моделей баз знаний, соответствующих методике, предполагающей взаимодействие всех разработчиков с семантической памятью и наличием иерархии администраторов. Если класс действия не является ключевым sc-элементом данного раздела, то данный класс действия не будет добавлен в результат работы агента.
