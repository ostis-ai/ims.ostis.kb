Под атомарным многократно используемым компонентом OSTIS подразумевается компонент, который в текущем состоянии библиотеки компонентов рассматривается как неделимый, то есть не содержит в своем составе других компонентов, представленных в какой-либо из библиотек компонентов в рамках IMS. В общем случае атомарный компонент может перейти в разряд неатомарных в случае, если будет принято решение выделить какую-то из его частей в качестве отдельного компонента. Все вышесказанное, однако, не означает, что даже в случае его платформенной независимости, атомарный компонент всегда хранится в sc-памяти как сформированная sc-структура. Например, платформенно-независимая реализация sc-агента всегда будет представлена набором scp-программ, но будет атомарным многократно используемым компонентом OSTIS в случае, если ни одна из этих программ не будет представлять интереса как самостоятельный компонент.
