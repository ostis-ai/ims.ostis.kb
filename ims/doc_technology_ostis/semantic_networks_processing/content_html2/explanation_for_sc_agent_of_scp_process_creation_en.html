Problem of the sc-agent of scp-process creation is creation of the scp-processes, corresponding to an scp-program. This sc-agent activated by the appearance in the sc-memory an question of initialization, belonging to the class of requests of interpretation of the scp-program.

After chacking initiation conditions creation of scp-process is executed by sc-agentfrom the accounting of specific parameters of interpretation scp-program, then search of an initial operator' is carried out of scp-process and its activization .

Creation of unique copies of the executed scp-program in case of each its start requires additional expenses of time and memory, however such approach has a row of essential advantages:

  1. creating scp-process can be reconfigured directly in progress, with no risk of damage to the original scp-program;
  2. implementation of the created scp-process does not require calls to the original scp-program, thus eliminating the need to synchronize access scp-processes,corresponding to the same scp-program as a sharing resource;
  3. execution of created scp-processes can be physically implemented on different devices in parallel, which greatly increases the effect of increasing the hardware basis for additional resources;
  4. there is no need to specify scp-process,within which some scp-variable' has a given value*, because each scp-variable' becomes unique within sc-memory as a result of copying also can have only one value*;
  5. when scp-process has been created scp-variables', corresponding to the in-parameters' of an scp-programare physically replaced with sc-elements, specified as parameters in the corresponding query interpretation.Such changeover allows to exclude additional operation of search value* for the specified scp-variables', as they become scp-constants'.
